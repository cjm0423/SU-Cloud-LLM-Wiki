## 📝 Summary

SU Cloud를 실제 학교 자원 위에서 시작하기 위한 전체 팀 킥오프.
OpenStack 수동 설치 → Proxmox 기반 실습 환경 → native Ubuntu 기반 production-like 구성으로 넘어가는 3단계 로드맵이 확정됐다.

**핵심 판단:** "바로 운영계를 만들기보다 학생들이 먼저 OpenStack 수동 설치와 packet flow를 이해해야 한다."

## 💬 주요 논의 내용

### 서버/자원 현황
- 교수님 제공 PC 2대: 소형(개발계/실습계 후보) + 대형(운영계 후보)
- 한 대 중심 구조 → HA/DR은 현실적으로 어려움
- 초기 목표: backup + 개발계/운영계 분리 + 최소 production-like 운영

### OpenStack 설치 전략
- **Kolla Ansible 바로 X** → 수동 설치로 controller/compute, provider/self-service network, packet flow를 먼저 이해
- 실습 환경: Proxmox 위에 VM 여러 개를 만들어 반복 실습
- 이후 병목 확인 → native Ubuntu/OpenStack 구성으로 전환 검토

### 네트워크 학습 블록 (필수)
- VM 트래픽이 bridge → router → DHCP → VXLAN tunnel → physical network를 어떻게 지나가는지 직접 추적
- AWS VPC처럼 논리적으로 같은 네트워크여도 물리적으로 다른 node에 있을 수 있음을 이해

### 원격 접속 모델
- **Proxmox + Tailscale** → 외부에서 학교 장비 접근하여 VM 생성/실습 가능
- 공용 계정, VPN, 원격 접속 정책은 추가 확인 필요

### AI 시대의 커리어 메시지 (전체 공유)
> "코드 작성 비용이 낮아지는 흐름에서, 무엇을 만들지 정의하고, 인프라 아키텍처를 이해하며, agent와 사람을 관리하는 역량이 더 중요해진다. DevOps/SRE도 자동화의 영향을 받기 때문에 Platform Engineering + Self-Service Platform 경험이 핵심."

### SU Cloud 가치 제안
- 단순 OpenStack 설치나 졸업작품이 아닌 **학교 내부 사용자에게 VM과 컴퓨팅 파워를 제공하는 준프로덕션 클라우드 플랫폼**
- 초기 고정 트래픽 후보: 연구실, 클라우드 컴퓨팅 수업, AI 실험 수요
