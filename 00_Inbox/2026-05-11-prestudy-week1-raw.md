# 1주차(05/11 ~ 05/17)

## 1. 클라우드 컴퓨팅 & OpenStack 개요

- **클라우드 컴퓨팅**: 인터넷을 통해 원격으로 컴퓨팅 자원 및 서비스를 제공하며, 기존 온프레미스 인프라의 비용과 유지보수, 확장성 문제를 해결함.
- **IaaS (Infrastructure as a Service)**: 가상화된 서버, 스토리지, 네트워크 등 IT 인프라를 서비스 형태로 제공함. 사용자는 인프라를 직접 구축할 필요 없이 필요한 자원만 선택해 사용 가능함.
- **OpenStack**: IaaS 클라우드 시스템을 구축하는 가장 대표적인 오픈소스 소프트웨어임.
    - **활용 사례**: 카카오, 블리자드, 라인 등 글로벌 기업들이 자체 클라우드 구축 시 표준처럼 활용함.
    - **역사**: 2010년 7월, NASA의 Nebula 프로젝트와 Rackspace의 기술이 합쳐져 시작된 프로젝트임.

## 2. 주요 아키텍처 특징

- **100% Python 개발**: 인프라 엔지니어가 직접 원인을 파악하고 코드를 수정하기 용이하도록 학습과 수정이 빠른 파이썬으로 개발됨. + 스크립트
- **MSA (Micro Service Architecture)**: 기능별로 분리된 작은 서비스들이 독립적으로 배포 및 확장 가능하며, 각 서비스는 API를 통해 유기적으로 통신함.

## 3. 핵심 컴포넌트 상세 (7대 요소)

| 컴포넌트명 | 역할 | 설명 |
| --- | --- | --- |
| **Keystone** | **Identity** | 전체 서비스의 인증 및 권한 관리를 담당하는 중앙 허브. |
| **Nova** | **Compute** | 가상 머신(VM) 인스턴스의 생성과 수명 주기를 관리함. |
| **Neutron** | **Networking** | 가상 네트워크, IP 관리, 라우팅 등 네트워크 인프라 제공. |
| **Glance** | **Image** | VM 생성에 필요한 OS 디스크 이미지 보관 및 관리. |
| **Cinder** | **Block Storage** | VM에 추가로 연결하는 고성능 블록 스토리지 서비스. |
| **Swift** | **Object Storage** | 비정형 데이터를 저장하는 대규모 확장성 오브젝트 스토리지. |
| **Horizon** | **Dashboard** | 오픈스택 자원을 GUI 기반으로 관리하는 웹 대시보드. |

## 4. 노드 구성 및 아키텍처

- **Controller Node**: 전체 클라우드 관리 및 웹 인터페이스를 제공하는 서버로, 높은 CPU 클럭과 적정한 메모리가 필요함.
- **Compute Node**: 실제 유저의 VM이 생성되어 동작하는 서버이며, 많은 CPU 코어와 대용량 메모리가 필수적임.
- **일반적인 구성**: 컨트롤러 1대 + 컴퓨트 N대 방식이 기본이며, 무중단 서비스를 위한 고가용성(HA) 환경에서는 컨트롤러를 3대 이상으로 구성함.

오픈스택 동작 구성

- 실제 인프라

```bash
[ 외부 인터넷 / 관리자 (클라이언트) ]
                     │
                     ▼ (API 호출 및 웹 접속)
╔════════════════════════════════════════════════════════╗
║             컨트롤러 노드 (Controller Node)            ║
║  - 클라우드의 중앙 제어탑 (Control Plane)              ║
╠────────────────────────────────────────────────────────╣
║  ▪ Horizon   : 웹 대시보드 (사용자 UI)                 ║
║  ▪ Keystone  : 통합 인증 및 권한 검증                  ║
║  ▪ Nova API  : 컴퓨팅 자원 스케줄링 및 제어 명령       ║
║  ▪ Neutron   : 가상 네트워크 구성 명령                 ║
║  ▪ Cinder    : 스토리지 볼륨 할당 명령                 ║
║  ----------------------------------------------------  ║
║  * Message Queue (RabbitMQ) : 각 노드 간 명령 전달망   ║
║  * Database (MariaDB)       : 전체 상태 및 설정 저장   ║
╚══════════════════════╦═════════════════════════════════╝
                       │
     (내부 관리 네트워크를 통한 제어 명령 하달)
                       │
                       ▼
╔════════════════════════════════════════════════════════╗
║             오픈스택 데이터센터 자원 풀 (Pool)         ║
╠══════════════════════╦═════════════════════════════════╣
║ 네트워크 노드(Network)║ 스토리지 노드 (Storage)        ║
║ - 외부 망과 VM 연결    ║ - 대규모 데이터 저장소         ║
║ ──────────────────── ║ ────────────────────────────── ║
║ ▪ L3 Router / NAT    ║ ▪ Cinder (블록 스토리지/볼륨)  ║
║ ▪ DHCP (IP 자동할당)   ║ ▪ Swift (오브젝트 스토리지)    ║
╚══════════╦═══════════╩════════════════╦════════════════╝
           │                            │
           │ (가상 네트워크 연결)       │ (가상 디스크 부착)
           ▼                            ▼
╔════════════════════════════════════════════════════════╗
║              컴퓨트 노드 (Compute Node)                ║
║  - 실제 가상 머신(VM)이 생성되고 구동되는 물리 서버    ║
╠════════════════════════════════════════════════════════╣
║  ┌─────────────────┐      ┌─────────────────┐          ║
║  │ [ 가상 머신 A ] │      │ [ 가상 머신 B ] │          ║
║  │ - vCPU / vRAM   │      │ - vCPU / vRAM   │          ║
║  │ - 가상 IP 할당  │      │ - 가상 IP 할당  │          ║
║  │ - 부착된 볼륨   │      │ - 부착된 볼륨   │          ║
║  └─────────────────┘      └─────────────────┘          ║
║ ────────────────────────────────────────────────────── ║
║        가상화 계층 (Hypervisor : KVM / QEMU 등)        ║
║ ────────────────────────────────────────────────────── ║
║          물리적 하드웨어 (물리 CPU, RAM, NIC)          ║
╚════════════════════════════════════════════════════════╝
```
